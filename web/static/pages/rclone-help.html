FileBot Admin

Rclone Setup Guide

Connect your cloud storage to FileBot using rclone.

Supported clouds

FileBot supports these rclone remotes (more can be added later):

  • de>gdrive – Google Drive
  • de>onedrive – Microsoft OneDrive
  • de>dropbox – Dropbox
  • de>mega – MEGA
  • de>s3 – S3 compatible storage

Step 1: Install rclone

On your server or local machine, install rclone from the official site:

https://rclone.org/downloads/

Step 2: Create a remote

Run the rclone config wizard and create a remote for your cloud.

rclone config

Follow the prompts to add a new remote. At the end you will have a remote name, for example:

gdrive
onedrive
dropbox
mega
s3

Step 3: Get the config

Locate your rclone config file. Common locations:

  • Linux: de>~/.config/rclone/rclone.conf
  • Windows: de>%USERPROFILE%\.config\rclone\rclone.conf
  • macOS: de>~/.config/rclone/rclone.conf

Open the file and copy the full block for the remote you want FileBot to use. It will look like:

[gdrive]
type = drive
client_id = ...
client_secret = ...
token = {...}
team_drive = ...

Step 4: Add in admin panel

In the FileBot admin panel:

  1. Open de>./admin.html and click Add rclone config.
  2. Select the cloud: de>gdrive, de>onedrive, de>dropbox, de>mega or de>s3.
  3. Paste the rclone remote block from de>rclone.conf into the credentials field.
  4. Choose who can use this config:
    • de>Free – for free users
    • de>Pro – for pro users
  5. Set Max users for this config (for example: 10).
  6. Save the config and, if needed, add more remotes with the + button.

Step 5: Test and use

After adding at least one active rclone config:

  • Send a small file to the bot as a test.
  • If no rclone config is available, the bot will show a maintenance message and will not accept files.
  • Once setup is correct, files will be uploaded via your configured cloud storage.

Troubleshooting

  • If uploads fail, re-check the rclone credentials you pasted.
  • Make sure the remote name in de>rclone.conf matches the cloud type you chose.
  • Ensure your server can reach the cloud provider (no firewall or proxy issues).
