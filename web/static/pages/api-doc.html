Overview

Base URL: de class="mono-inline" id="api-base-url"> https://your-domain.com

Use the endpoints below to fetch watch/download links or build your own dashboard.

Authentication

Admin endpoints require a JWT token in the de class="mono-inline">Authorization header.

Request One-Time Code

de class="mono-inline">POST /auth/request-code

Send your Telegram User ID to receive a one-time code in the bot.

Verify Code & Get Token

de class="mono-inline">POST /auth/verify-code

Verify the code and receive a JWT token to use with admin endpoints.

Public Endpoints

Watch File

de class="mono-inline">GET /watch/{file_id}

Opens a streaming page for the given file_id.

Download File

de class="mono-inline">GET /download/{file_id}

Downloads the file if available and not expired.

My Files (optional)

de class="mono-inline">GET /api/user/myfiles

Returns the list of files for the authenticated user (via token/OTP). Response shape (example): 

{
  "user_id": 123456789,
  "files": [
    {
      "file_id": "abcd1234",
      "name": "video.mp4",
      "start_id": "start_abc123",
      "size": 1073741824,
      "size_human": "1.0 GB",
      "created_at": "2025-12-01T10:00:00Z",
      "expired": false,
      "deep_link": "https://t.me/YourBot?start=start_abc123"
    }
  ]
}

Admin Endpoints

All admin endpoints require: de class="mono-inline">Authorization: Bearer <JWT_TOKEN>

Dashboard

de class="mono-inline">GET /api/admin/dashboard

Returns statistics for the admin dashboard (users, tasks, storage, etc.).

Users

de class="mono-inline">GET /api/admin/users?skip=0&limit=10

Get paginated list of users with their limits and status.

Config

de class="mono-inline">GET /api/admin/config

Reads configuration such as plans, limits, channels.

Logs

de class="mono-inline">GET /api/admin/logs?skip=0&limit=50

Fetches recent log entries for debugging.

Site Config

de class="mono-inline">POST /api/admin/site-config

Update public site config (title, logo, colors, texts).

Rate Limits

API requests are limited per IP and per token to protect the service. Use caching and avoid unnecessary polling.
